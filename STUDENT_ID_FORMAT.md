# Student Registration Number Format Guide

## Overview
Students in the PICT Guard system use **institution-assigned PRN (Permanent Registration Number)** as their unique identifier. These are NOT auto-generated by the system.

## Why PRN Instead of Auto-Generation?

Each student already has a unique permanent registration number (PRN) assigned by the institution. Using this approach:
- ✅ Matches institutional records exactly
- ✅ Prevents duplicate registrations
- ✅ Simplifies record reconciliation
- ✅ Maintains consistency with existing systems

## Excel Format

### Column Structure (6 columns required)

| Column | Field | Format | Example | Required |
|--------|-------|--------|---------|----------|
| 1 | RegNo/PRN | Text (Unique) | 12101001 | ✅ YES |
| 2 | Name | Text | Rahul Sharma | ✅ YES |
| 3 | Email | Email format | rahul.sharma@pict.edu | ✅ YES |
| 4 | Mobile | 10 digits (no spaces) | 9876543210 | ✅ YES |
| 5 | DOB | YYYY-MM-DD | 2003-03-15 | ✅ YES |
| 6 | Year | 1, 2, 3, or 4 | 3 | ✅ YES |

### Sample Entry
```
RegNo/PRN: 12101001
Name: Rahul Sharma
Email: rahul.sharma@pict.edu
Mobile: 9876543210
DOB: 2003-03-15
Year: 3
```

## Database Handling

### How System Processes Student Records

```
Excel Input (Column 1)
         ↓
    RegNo/PRN
         ↓
  Duplicate Check:
  - By RegNo (primary)
  - By Email
  - By Mobile
         ↓
   If unique → Save to DB
   If duplicate → Reject with error
```

### Duplicate Detection Rules

The system rejects student uploads if any of these exist:
1. **RegNo/PRN already exists** (highest priority)
2. **Email already registered**
3. **Mobile number already registered**

This prevents:
- Same student registered twice
- Email conflicts
- Mobile number conflicts

## Important Rules

### ✅ DO's
- ✅ Use your institution's official PRN numbers
- ✅ Ensure each PRN is UNIQUE
- ✅ Keep format consistent (12101001, not 121-01-001)
- ✅ Use 10-digit mobile numbers
- ✅ Use YYYY-MM-DD for dates
- ✅ Update the Excel template with current year values

### ❌ DON'Ts
- ❌ Don't auto-generate or make up PRN numbers
- ❌ Don't use duplicate PRNs, emails, or mobiles
- ❌ Don't leave any required field empty
- ❌ Don't use special characters in PRN
- ❌ Don't manually edit IDs after upload
- ❌ Don't change the column order

## Troubleshooting

### Error: "RegNo/PRN already exists"
**Cause:** Student with this PRN already registered
**Solution:** Check institution records, verify PRN is correct, or use different PRN

### Error: "Email already registered"
**Cause:** Email address belongs to another student
**Solution:** Verify email is unique, check for duplicate registrations

### Error: "Mobile number already registered"
**Cause:** Mobile number belongs to another student
**Solution:** Verify phone number is correct

### Error: "Invalid email format"
**Cause:** Email doesn't follow standard format
**Solution:** Use format: name@domain.com (e.g., rahul.sharma@pict.edu)

### Error: "Invalid mobile number"
**Cause:** Mobile number not exactly 10 digits
**Solution:** Use 10-digit format without spaces, dashes, or country codes

### Error: "Invalid date format"
**Cause:** DOB not in YYYY-MM-DD format
**Solution:** Convert date to YYYY-MM-DD (e.g., 2003-03-15 for March 15, 2003)

## System Behavior

### Student Record Lifecycle
1. **Excel Submission**: Admin uploads Excel file via bulk upload endpoint
2. **Validation**: System checks all 6 columns are present
3. **Duplicate Check**: System verifies RegNo, Email, Mobile are unique
4. **Storage**: If all checks pass, student record created
5. **Verification**: Records searchable by RegNo or Email
6. **Validity**: Valid until end of academic year (set during upload)

### Search & Lookup
- Students can be searched by: RegNo, Name, Email, or Mobile
- Exact match on RegNo returns specific student
- Partial match on Name returns list of students
- Used by: Alumni, Visitor, Faculty forms for verification

## Comparison: Faculty vs. Student ID Format

| Aspect | Faculty | Student |
|--------|---------|---------|
| ID Format | PICT-FAC-XXX (auto-generated) | Institution PRN (manual entry) |
| ID Source | System counter | Excel upload |
| Uniqueness | Auto-ensured | Must be provided correctly |
| Duplicates | Impossible (auto-generated) | Possible (if PRN entered twice) |
| Example | PICT-FAC-001 | 12101001 |
| Upload Method | Bulk Excel (optional) | Bulk Excel (required) |
